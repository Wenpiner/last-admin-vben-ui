# 爬虫客户端管理模块

## 📋 功能概述

爬虫客户端管理模块提供了完整的爬虫客户端信息管理功能，支持查看客户端状态、性能指标和修改配置。

## 🚀 主要功能

### 1. 爬虫客户端列表显示
- ✅ 分页表格展示爬虫客户端信息
- ✅ 显示客户端基本信息（名称、ID、主机名、IP地址等）
- ✅ 实时状态标识（在线/离线/繁忙/空闲）
- ✅ 性能指标展示（CPU、内存、磁盘使用率）
- ✅ 任务统计信息（当前任务数、已完成任务数、已处理URL数等）
- ✅ 相对时间显示（最后心跳时间显示为"X分钟前"，鼠标悬停显示完整时间）

### 2. 爬虫客户端搜索
- ✅ 按客户端ID搜索
- ✅ 按状态筛选

### 3. 爬虫客户端管理
- ✅ 新增客户端（客户端ID、名称、主机名、IP地址、端口、最大并发任务数、标签）
- ✅ 编辑客户端信息（名称、主机名、IP地址、端口、最大并发任务数、标签）
- ✅ 删除客户端（带确认对话框）
- ✅ 查看客户端详细信息
- ✅ 手动刷新客户端状态
- ✅ 自动刷新功能（10秒间隔，可通过开关控制）

## 🔧 技术实现

### API接口
- `POST /iras-api/api/v1/crawler/clients/list` - 获取爬虫客户端列表
- `POST /iras-api/api/v1/crawler/clients/modify` - 创建或修改爬虫客户端（根据是否有id判断）
- `DELETE /iras-api/api/v1/crawler/clients/{id}` - 删除爬虫客户端

### 数据结构
```typescript
interface ClientInfo {
  id?: number;
  name: string;                    // 客户端名称
  clientId: string;                // 客户端ID
  hostname: string;                // 主机名
  ip_address: string;              // IP地址
  port: number;                    // 服务端口
  status: string;                  // 客户端状态
  max_concurrent_tasks: number;    // 最大并发任务数
  current_tasks: number;           // 当前执行的任务数
  cpu_usage: number;               // CPU使用率(%)
  memory_usage: number;            // 内存使用率(%)
  disk_usage: number;              // 磁盘使用率(%)
  network_rx: number;              // 网络接收速率(KB/s)
  network_tx: number;              // 网络发送速率(KB/s)
  last_heartbeat: string;          // 上次心跳时间
  last_task_finished: string;      // 上次任务完成时间
  total_tasks_completed: number;   // 已完成任务总数
  total_urls_processed: number;    // 已处理URL总数
  total_errors: number;            // 错误总数
  version: string;                 // 客户端版本
  capabilities: string;            // 客户端能力(JSON格式)
  error_message: string;           // 最近的错误消息
  tags: string;                    // 标签(逗号分隔)
  createdAt?: string;              // 创建时间
  updatedAt?: string;              // 更新时间
}
```

### 特殊功能实现

#### 1. 相对时间显示
最后心跳时间使用相对时间格式显示（如"5分钟前"），提供更直观的时间感知：
- 小于1分钟：显示"X秒前"
- 1-60分钟：显示"X分钟前"
- 1-24小时：显示"X小时前"
- 1-30天：显示"X天前"
- 超过30天：显示完整日期时间

鼠标悬停在时间上会显示完整的时间戳。

#### 2. 自动刷新功能
- 页面右上角提供自动刷新开关
- 开启后每10秒自动刷新一次客户端列表
- 组件卸载时自动清理定时器，避免内存泄漏
- 用户可随时开启或关闭自动刷新

## 🎯 使用方式

### 1. 访问爬虫客户端管理
导航到 `iRas 管理 > 爬虫客户端管理` 页面

### 2. 查看爬虫客户端列表
- 页面加载时自动显示所有爬虫客户端的分页列表
- 使用搜索表单筛选客户端
- 点击刷新按钮更新客户端状态
- 开启自动刷新开关实现定时更新

### 3. 新增爬虫客户端
1. 点击页面右上角的 "新增爬虫客户端" 按钮
2. 填写客户端信息：
   - 客户端ID（必填）
   - 客户端名称（必填）
   - 主机名（可选）
   - IP地址（可选）
   - 端口（可选，默认8080）
   - 最大并发任务数（必填，默认5，范围1-100）
   - 标签（可选，多个标签用逗号分隔）
3. 点击确认保存

### 4. 编辑爬虫客户端
1. 点击客户端行的 "编辑" 按钮
2. 修改客户端信息（客户端ID不可修改）
3. 点击确认保存

### 5. 删除爬虫客户端
1. 点击客户端行的 "删除" 按钮
2. 在确认对话框中确认删除操作

### 6. 监控客户端状态
- 查看实时状态标识（在线/离线/繁忙/空闲）
- 监控性能指标（CPU、内存、磁盘使用率）
- 查看任务统计信息
- 通过相对时间快速了解客户端最后活跃时间

## 📊 状态说明

- **在线 (Online)**: 客户端正常运行，可以接收任务
- **离线 (Offline)**: 客户端未连接或已停止
- **繁忙 (Busy)**: 客户端正在执行任务
- **空闲 (Idle)**: 客户端在线但没有执行任务

## ⚠️ 注意事项

1. **客户端ID唯一性**：新增客户端时，客户端ID必须唯一；编辑时客户端ID不可修改
2. **并发任务数限制**：最大并发任务数范围为 1-100
3. **端口范围**：端口号范围为 1-65535
4. **自动刷新**：建议在需要实时监控时开启自动刷新，不需要时关闭以减少服务器负载
5. **性能监控**：关注CPU、内存、磁盘使用率，避免资源耗尽
6. **删除确认**：删除操作不可恢复，请谨慎操作
7. **标签格式**：多个标签使用英文逗号分隔，如：`tag1,tag2,tag3`
